# Agent 是什么

**Agent = 一个持续存在的计算实体**

- 能接收输入
- 能做出决策
- 能执行动作
- 能在时间中保持连续性

Agent 不是生命，不是意识。

---

## 基本前提

- LLM 是函数，不是真正智能
- Agent 没有内生目标
- Agent 不主动行动，只响应指令或调度
- 一切设计以“能实现”为第一原则

---

## Agent 的最小完备结构

**Agent 只包含 6 个不可删除的功能。**

少一个不成立，多一个是冗余。

---

## 六个核心功能

**1. 认知（Cognition）**
**2. 记忆（Memory）**
**3. 行为调制状态（Control State）**
**4. 行动（Action / MCP）**
**5. 时间 / 调度（Runtime）**
**6. 感官（Perception）**

### 1. 认知（Cognition）

- 根据输入和状态生成意图
- 本质是函数（当前是 LLM）

---

### 2. 记忆（Memory）

- 保存历史信息
- 保证长期连续性

---

### 3. 行为调制状态（Control State）

- 非任务状态
- 用于整体行为控制
- 不等同于情绪或动机

---

### 4. 行动（Action / MCP）

- 执行外部动作
- 对接软件或硬件

---

### 5. 时间 / 调度（Runtime）

- 管理任务执行
- 支持中断与恢复
- 同一时刻只有一个执行流

---

### 6. 感官（Perception）

- 接收外部输入
- 输入必须结构化

---

## 明确不包含的内容

- 自我意识
- 情绪体验
- 主动目标
- 世界模型
- 类人智能

---

## 关于 KITT

- KITT 不是单个 Agent
- KITT = 多个 Agent 的系统
- 只有一个 Agent 直接与人类对话
- 其余 Agent 是功能专家

---

## 设计约束

> 任何新增能力，
> **必须能归入这 6 个功能之一。**
> 否则，设计有问题。

---

## 结论

**6 个功能 = 全部。**
这是 Agent 的最终定义。
