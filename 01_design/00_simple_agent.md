# Agent 是什么

> Agent = 一个持续存在的计算实体

- 能接收输入
- 能做出决策
- 能执行动作
- 能在时间中保持连续性

Agent 不是生命，不是意识。

---

## 基本前提

- LLM 是函数，不是真正智能
- Agent 没有内生目标
- Agent 不主动行动，只响应指令或调度
- 一切设计以“能实现”为第一原则

---

## Agent 的最小完备结构

**Agent 只包含 6 个不可删除的功能。**

少一个不成立，多一个是冗余。

---

## 六个核心功能

- **1. 感官（Perception）**
- **2. 认知（Cognition）**
- **3. 行动（Action / MCP）**
- **4. 时间 / 因果（Runtime）**
- **5. 记忆（Memory）**
- **6. 整体状态（Agent State）**

### 1. 感官（Perception）

- 接收外部刺激并进行无语义编码
- 输出统一、低语义的内部信号

---

### 2. 认知（Cognition）

- 将输入材料组织为可判定的问题空间
- 形成一次可审计的决策结果

---

### 3. 行动（Action / MCP）

- 将认知意图变成可执行的副作用操作
- 不做判断，只如实返回结果

---

### 4. 时间 / 因果（Runtime）

- 维护单一时间线与因果裁决
- 允许认知并发，但行动必须受时间线约束

---

### 5. 记忆（Memory）

- 保存长期事实与历史记录
- 不进行语义判断或自动整理

---

### 6. 整体状态（Agent State）

- 描述 Agent 的整体运行背景
- 派生于系统事实，只读不控

---

## 明确不包含的内容

- 自我意识
- 情绪体验
- 主动目标

---

## 关于 KITT

- KITT 不是单个 Agent
- KITT = 多个 Agent 的系统
- 只有一个 Agent 直接与人类对话
- 其余 Agent 是功能专家

---

## 设计约束

> 任何新增能力，
> **必须能归入这 6 个功能之一。**
> 否则，设计有问题。

---

## LLM 的系统定位

- LLM 不是 Agent
- LLM 不是大脑
- LLM 不持有状态
- LLM 不定义目标

LLM 在本系统中仅被视为一种**可调用的计算工具**：

- 可被 Cognition 使用，用于推理与判断
- 可被 Organize 使用，用于结构化输入材料
- 可被替换、限制或移除

> Agent 的决策一致性来自系统结构，
> 而不是来自任何单一模型的能力。

## 结论

**6 个功能 = 全部。**
这是 Agent 的最终定义。
