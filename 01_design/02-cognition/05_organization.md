## Organize 阶段定义（Draft）

### 1. 阶段定位

**Organize 是认知流程中最后一个非认知阶段。**

其职责是：

> 将 Collection 阶段产出的原始信息集合，
> 整理、规范并结构化为一次完整、封闭的 CognitionInput。

Organize **不进行认知推理**，
也**不参与决策或判断行为**。

---

### 2. 输入与输出

**输入：**

- Collection 阶段产出的原始信息集合（Collected Inputs）

**输出：**

- 单一的 CognitionInput 结构：

  ```
  CognitionInput {
    intent,
    context
  }
  ```

Organize 阶段的输出 **必须且仅能是 CognitionInput**。
任何其他形式的中间结果都不应暴露给下游。

---

### 3. 核心职责

Organize 阶段负责完成以下工作：

1. **信息整理与结构化**
   将非结构化或半结构化信息映射到系统已定义的 schema。

2. **信息合并与去重**
   对来自多个来源的重复或等价信息进行合并。

3. **摘要与压缩**
   在不引入新结论的前提下，对信息进行长度与复杂度压缩。

4. **Intent 生成**
   明确本次认知所需解决的单一问题或任务类型，并生成 intent。

5. **Context 构造**
   组织所有被允许进入认知层的事实性信息，形成封闭上下文。

---

### 4. 关于 LLM 的使用约束

Organize 阶段**允许使用 LLM**，
但仅限于承担信息整理与结构化职责。

在该阶段中，LLM 的角色被严格限定为：

> **信息组织器（Organizer），而非认知推理器（Thinker）。**

LLM 在 Organize 阶段中可以用于：

- 将非结构化输入映射为预定义 schema
- 从自然语言中显式生成 intent 结构
- 对信息进行去重、合并与无结论摘要
- 规范化不同来源的信息表达形式

LLM 在 Organize 阶段中 **不得被用于**：

- 推断缺失事实或补全未知信息
- 判断信息的重要性、正确性或优先级
- 解释信息之间的因果或意义
- 对后续行动或策略产生任何影响

所有通过 LLM 生成的 Organize 输出，
必须完全符合 CognitionInput 的结构定义，
否则视为 Organize 阶段失败。

---

### 5. 明确禁止的行为

Organize 阶段 **不得执行以下行为**：

- 判断信息的对错或真实性
- 决定信息的重要性或优先级
- 选择行动方案或策略
- 引入推断、假设或结论
- 替代认知层完成“理解”或“判断”

一句话约束：

> Organize 可以整理信息，但不能解释信息的意义。

---

### 6. 确定性与可回放性要求

Organize 阶段的输出必须满足：

- **封闭性**：不再依赖外部系统或隐式状态
- **可序列化**：可被完整记录与传输
- **可回放**：可在未来复现同一次认知输入
- **自洽性**：内部字段语义明确、无歧义

这些要求是认知层作为纯函数成立的前提。

---

### 7. 与 Cognition 层的边界

- Cognition 层只接受 CognitionInput
- Cognition 层不得反向请求补充输入
- Cognition 层不得修改 intent 或 context

一旦进入 Cognition 阶段，
认知边界即被视为已锁定。

---

### 8. 最小实现说明（非强制）

在最小实现中，Organize 阶段可以退化为：

- 直接从单一原始输入生成 intent
- 将有限的上下文信息直接封装为 context
- 不执行复杂合并或摘要逻辑

该退化不影响架构正确性，
但 Organize 概念在系统中必须显式存在。

---

### 9. 阶段边界声明（一句话版）

> Organize 阶段负责将系统已收集的信息，
> 整理为一次可被认知层直接使用的标准输入，
> 而不对这些信息做任何判断或决策。
