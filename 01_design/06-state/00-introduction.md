# Agent State · Introduction

**Agent State 描述的是 Agent 作为一个整体，在当前时刻所处的运行状态。**

它不是某个功能模块的内部状态，
也不是一个用于决策或控制的系统组件，
而是 Agent 在持续运行过程中形成的一组
**全局执行条件与约束的快照**。

在人类身上，也存在类似的整体状态：

一个人可能处于——
专注、疲劳、放松、紧张、被打断、睡眠不足、过载……

这些状态并不直接决定一个人“想什么”或“做什么”，
但它们会深刻影响：

- 是否能立刻行动
- 是否适合被打断
- 同样的刺激是否会被忽略
- 同样的决定是否会被推迟或降级执行

**Agent State 对 Agent 的意义，正是这种“整体运行状态”。**

---

Agent 并非始终以同一种方式运行。

在不同阶段、不同负载、不同任务背景下，
即使是**相同的认知结果**、**相同的动作请求**，
其可执行性、优先级与响应方式也可能完全不同。

这种差异并不来自新的判断或推理，
而是源于 Agent 当前所处的——

> **整体运行状态（Agent State）**

正如同一个人在“高度专注”和“极度疲劳”时，
面对同一句话、同一个请求，
其反应方式并不相同。

---

## 状态的性质

Agent State 具有以下基本特征：

- 它**属于 Agent 本身**，而非某个子系统
- 它是**横向的**，对 Cognition、Runtime、Action 等模块同时生效
- 它是**连续的、上下文相关的**，而非有限枚举状态
- 它不包含决策逻辑，也不进行推理
- 它通常被读取，而非被显式调用或切换

Agent State 不回答“应该做什么”，
它回答的是：

> 在当前这种“整体状态”下，
> **什么事情是自然发生的，
> 什么事情应当被延迟、抑制，
> 什么事情即使被提出也应当被忽略。**

这与人体状态的作用完全一致。

---

## 设计目的

引入 Agent State 的目的，
**不是为了模拟情绪、意识或主观体验**，
而是为了让 Agent 作为一个**持续存在的运行实体**，
在并发输入、跨时间行为与资源受限的条件下，
仍然保持行为的一致性与结构的可理解性。

在人类系统中，
“整体状态”天然存在，因此无需显式建模；
而在 Agent 系统中，
如果缺失这一层：

- 全局运行约束将被迫分散到各个模块内部
- 以条件判断、临时规则的形式隐式存在
- 系统结构会随复杂度增长而逐渐失真

Agent State 的作用，
正是将这些**原本隐含、分散、难以追踪的运行条件**
提升为一个清晰、统一、可演化的系统层。

---

这段 Introduction 的唯一目的仍然只有一个：

**明确这是 Agent 自己的状态，
就像人体的“状态”一样，
它影响一切，但不直接控制任何一个具体行为。**
