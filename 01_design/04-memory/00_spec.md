## Memory 系统规范（Memory Specification）

### 1. 定位

Memory 是 **Agent 的长期持久化存储系统**。

其唯一目的，是在时间维度上保存信息，以维持 Agent 的连续性与可追溯性。

Memory **不是智能系统**，不承担任何认知、推理或决策职责。

---

### 2. 核心原则

#### 2.1 非认知原则

Memory 不具备、也不允许具备以下能力：

- 推理
- 总结
- 关联
- 抽象
- 重要性判断
- 相关性判断
- 主动行为

Memory 仅存储原始记录，不对内容做任何语义处理。

---

#### 2.2 被动访问原则

Memory 不能被 Cognition 直接访问。

Memory 中的内容只能通过 **Collection 阶段**，
在明确、可配置、可审计的规则下，被引入一次具体的认知过程。

访问路径固定为：

```
Memory → Collection → Cognition
```

Memory 不感知访问意图，也不感知访问结果。

---

#### 2.3 有界访问原则

Memory 在**认知可见性层面**必须是有限的。

- Memory 的物理规模可以增长
- 但任意一次认知过程：

  - 只能访问一个**明确、有界的 Memory 子集**

Memory 不负责决定取多少、取哪些内容。

---

### 3. 存储内容范围

Memory 只允许存储以下三类信息：

#### 3.1 事实（Facts）

- 稳定、可重复引用
- 与具体认知过程弱相关

例如：配置、身份信息、确认后的规则。

---

#### 3.2 历史（History）

- 已发生事件的顺序记录
- 只追加，不隐式修改

例如：指令记录、执行结果、状态变更日志。

---

#### 3.3 状态（State）

- 描述 Agent 的长期运行状态
- 表示某一时刻的系统快照

例如：当前模式、长期任务进度、持久标志位。

---

Memory 不存储“知识”“理解”“结论推导过程”等认知产物。

---

### 4. Agent 私有性

Memory 是 **Agent 私有资源**。

- 每个 Agent 拥有独立的 Memory
- Memory 默认不共享
- 不作为跨 Agent 通信、同步或知识共享机制

任何共享信息，必须通过独立系统或显式接口完成。

---

### 5. 时间语义

Memory 仅记录时间戳，不理解时间含义。

Memory 不具备以下概念：

- 最近
- 重要
- 过期
- 高频
- 异常

所有时间相关的判断均属于 Memory 之外的系统。

---

### 6. 有限容量原则

Memory 不是无限存储。

- Memory 必须配置明确的容量上限
- 当达到容量限制时：

  - Memory **拒绝新的写入请求**
  - 不进行自动删除、压缩、合并、总结或重排

Memory 不负责决定哪些内容可以被丢弃。

---

### 7. 重组与清理约束

Memory **不提供内部重组能力**。

Memory 不允许包含以下行为：

- 自动清理
- 自动归档
- 自动压缩
- 自动总结
- 自我整理

任何 Memory 的裁剪、迁移、归档或重建行为：

- 必须由外部系统以**显式流程**完成
- 以整体替换或明确操作的形式发生
- Memory 本身仅作为被操作对象存在

---

### 8. 边界判定规则（防设计退化）

如果某个功能需要回答以下任一问题：

- “这个重要吗？”
- “这个是否与当前任务相关？”
- “哪些内容可以被删除？”
- “哪些内容值得长期保留？”
- “是否应该暴露给 Cognition？”

该功能 **不应存在于 Memory 系统中**。

---

### 9. 总结性定义

Memory 是一个：

- 冷态的
- 被动的
- 无语义的
- 有容量上限的
- Agent 私有的
- 仅负责时间连续性的

**长期存储容器**。

它的价值不在于优化认知，
而在于为认知系统提供一个**确定、受控、可审计的历史基础**。
