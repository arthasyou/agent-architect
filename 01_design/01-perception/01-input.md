### Input · Perception 的输入定义

在当前阶段，Agent 处于纯软件形态，但 Perception 的输入定义必须面向完整目标形态：
**单体可运行、未来可接硬件、最终可多 Agent 协作，并将人类指令视为核心输入之一。**

因此，Perception 的输入不以“硬件/软件”划分，而以**“是否来自 Cognition 内部”**划分：

> **凡是来自认知系统边界之外的信息，都属于 Perception 的输入。**

这一定义覆盖并统一以下三类关键输入来源：

1. **人类指令输入（Human Instruction）**
   人类以自然语言或其他交互方式输入的指令，是最核心、最优先被系统支持的输入类型。
   Perception 在此只负责“接收与转入系统”，不负责解释其意图或做任务拆解。

2. **软件世界输入（Software Environment）**
   来自操作系统、应用、网络、文件、日志、消息队列等的软件事件与数据流。
   这些输入形态高度异构，可能连续或离散、结构化或非结构化、可靠或噪声很高。
   Perception 不对其语义作任何预设，只承认它们作为“外界刺激”进入系统。

3. **外部实体输入（External Entities）**
   包括但不限于：

- **硬件/传感器输入（未来接口）**：摄像头、麦克风、温度、加速度等
- **其他 Agent 的输出（Multi-Agent 协作）**：当多个 Agent 协作时，其他 Agent 对外发出的结果、事件或请求，对本 Agent 而言属于“外部刺激”，应通过 Perception 进入，而不是直接注入 Cognition。

在输入层面，Perception 只做最低限度的假设：

- 输入可能来自任何外部实体或系统
- 输入形态不统一、节奏不统一、可靠性不统一
- 输入不等于“已具备意义的信息”，只是“刺激发生了”
- Perception 必须允许新增输入来源而不改变认知层边界（为硬件与多 Agent 留接口）

Perception 的职责不是筛选“有用输入”，也不是理解输入含义；
它的职责是**承认外界刺激的进入**，并为后续的统一编码与传递建立前提。
